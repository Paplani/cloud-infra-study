# HTTP / HTTPS / SSH 개념 및 실습

## 1. 실습 목적

HTTP와 HTTPS의 차이를 이해하고,  
포트(80, 443, 22)가 실제로 어떤 역할을 하는지 확인했다.

또한 SSH 접근 방식과 보안 설정의 중요성을 함께 정리했다.

이번 실습의 핵심은  
**웹 접속과 서버 접근이 각각 어떤 방식으로 이루어지는지 이해하는 것**이다.

- Cloud: AWS EC2

- Web Server: nginx
- Access:
  - SSH
  - HTTP (브라우저)
  - curl

## 3. 핵심 개념

### HTTP (80)

HTTP는 웹 브라우저와 서버가 데이터를 주고받는 기본 프로토콜이다.

- 포트: 80
- 특징: 암호화 없음 (평문 통신)

즉, 중간에서 데이터를 볼 수 있기 때문에 보안에 취약하다.

---

### HTTPS (443)

HTTPS는 HTTP에 암호화를 추가한 방식이다.

- 포트: 443
- 특징: SSL/TLS 기반 암호화

데이터가 암호화되어 전송되기 때문에  
로그인 정보나 개인정보를 보호할 수 있다.

단, HTTPS를 사용하려면 다음이 필요하다.

- SSL 인증서
- 서버 설정 (nginx 등)
- 443 포트 오픈

---

### SSH (22)

SSH는 서버에 원격으로 접속하기 위한 프로토콜이다.

- 포트: 22
- 용도: 서버 관리 (로그인, 명령 실행 등)

웹 서비스와 달리, SSH는 관리자만 접근해야 한다.

---

## 4. 포트와 역할 정리

| 포트 | 용도 | 설명 |
|------|------|------|
| 22 | SSH | 서버 원격 접속 |
| 80 | HTTP | 웹 접속 |
| 443 | HTTPS | 보안 웹 접속 |

---

## 5. Security Group 설정 기준

### HTTP / HTTPS

```text
Source: 0.0.0.0/0
```

- 모든 사용자 접근 허용
- 웹 서비스는 외부 공개가 목적

---

### SSH

```text
Source: 내 IP/32
```

- 관리자만 접근 허용
- 보안상 전체 공개는 위험

---

👉 정리

- 웹(80, 443) → 전체 공개
- SSH(22) → 제한된 접근

---

## 6. 실습 내용

### 6-1. HTTP 접속 확인

```text
http://<public-ip>
```

- nginx 페이지 정상 출력

---

### 6-2. HTTPS 접속 시도

```text
https://<public-ip>
```

결과:

- 접속 실패 또는 경고 발생

이유:

- SSL 인증서 없음
- nginx HTTPS 설정 없음

---

### 6-3. 443 포트 추가 (Security Group)

- HTTPS (443) 추가
- Source: 0.0.0.0/0

→ 여전히 접속 불가

👉 포트만 열어서는 HTTPS가 동작하지 않음을 확인

---

### 6-4. SSH 접근 확인

```bash
ssh -i my-key.pem ubuntu@<public-ip>
```

정상 접속 확인

---

## 7. 확인 결과

- HTTP는 별도 설정 없이 바로 사용 가능
- HTTPS는 인증서와 서버 설정이 필요
- 포트만 열어도 HTTPS는 동작하지 않음
- SSH는 서버 운영을 위한 필수 접근 방식

---

## 8. 운영 관점에서 중요한 점

- HTTP는 간단하지만 보안에 취약하다
- 실제 서비스는 대부분 HTTPS를 사용한다
- SSH는 반드시 접근 범위를 제한해야 한다

특히 SSH를 전체 공개하면  
외부 공격 시도가 많아질 수 있기 때문에  
**내 IP만 허용하는 방식이 더 안전하다**

---

## 9. 발생할 수 있는 문제

### HTTPS 접속 실패

가능한 원인:

- SSL 인증서 없음
- nginx 설정 없음
- 443 포트 미허용

---

### SSH 접속 문제

가능한 원인:

- 22 포트 차단
- pem 파일 권한 문제
- 사용자명 오류

---

## 10. 트러블슈팅

### HTTPS가 안 될 때

1. 443 포트 열려 있는지 확인
2. nginx HTTPS 설정 여부 확인
3. 인증서 존재 여부 확인

👉 포트만 열어서는 해결되지 않음

---

### SSH 접속 안 될 때

1. Security Group에서 22 포트 확인
2. pem 파일 권한 확인

```bash
chmod 400 my-key.pem
```

3. 사용자명 확인 (ubuntu)

---

## 11. 배운 점

- HTTP와 HTTPS는 보안 수준에서 큰 차이가 있다
- HTTPS는 단순 포트 문제가 아니라 인증서와 설정이 필요하다
- SSH는 서버 운영에 필수지만 보안 설정이 매우 중요하다
- 포트마다 역할이 다르기 때문에 목적에 맞게 설정해야 한다
